# 更新日志 (Changelog)

## [未发布]

### 2026-07-23 · 🎮 远征情境交互重做

- 删除固定右侧“远征战术终端”，战斗画布恢复全宽；出发整备与行动结算改为居中面板。
- 搜索与营地改为仅在需要决策时出现的底部情境操作条，战斗和撤离守点只保留核心战斗 HUD。
- 路线列表改为顶部目标条，点击或按 `M` 在路线、事件和撤离点之间轮换追踪；靠近地点仍统一按 `E` 交互。
- 顶部背包状态改为按需抽屉，支持 `B` 开关和安全袋整理；旧活动存档的待选物品会强制打开取舍卡。
- 增加 `R` 补给快捷键，释放 `Tab` 的默认焦点导航；退出统一使用 `Esc` 风险确认，不再提供终端内的“放弃本局”。
- 170 项 Node 自动化测试和 31 项浏览器集成检查全部通过。完整说明见 `doc/design/expedition-context-interaction-20260723.md`。

### 2026-07-23 · 🧭 远征核心玩法精简

- 武器改为出发前选择、局内锁定；三种武器基础强度重新校准，移动只扩大散布，怪物生命不再随玩家等级反向膨胀。
- 搜索收拢为“快速拿取 / 彻底搜刮”两种一次性决策，宠物侦察作为被动加成；满包时自动保留更有价值的未保护物品。
- 风险收拢为 0～100 的单一警戒值，不再展示超限或返程压力；撤离离圈只暂停进度，增援继续但进度不回退。
- 局内仅保留队长宠物的一个主动队伍技能；取消无限自然回血，恢复属性改为强化补给治疗。
- 战斗或撤离守点中退出按战败结算，安全行进时才可止损；退出确认与风险预估会明确显示对应损失及暴露配装。
- 顶部 HUD 仅保留清理进度、生命、警戒和背包等核心信息；后续情境交互改版已彻底移除战术终端。
- Node 自动化测试与浏览器集成检查全部通过。完整说明见 `doc/design/expedition-gameplay-simplification-20260723.md`。

### 2026-07-21 · 🎯 参考搜打撤玩法第一阶段

- 主角战斗改为手动瞄准固定弹道，加入步枪、霰弹枪、精确枪的独立弹匣、备弹、换弹、散布、射程和移动射击惩罚；敌方远程攻击改为可躲避、可被掩体阻挡的弹丸，并增加 Canvas 准星。
- 搜索点改为 2～3 个实体容器和可中断读条；完成前不预结算，移动、受伤或取消会释放容器，进度与容器状态支持活动局存档。
- 增加深区应急撤离点，与入口撤离分别校验深度、补给、守点时长、敌人与增援压力；失败只回收显式保险物品。
- 新增局外仓库、待领取箱、真实配装、五类合约和跨重启幂等结算账本；战利品入仓后不再同时自动折现，护甲、宠物链接器与消耗品均已接入局内效果和失败遗失。
- 存档升级至 `1.5.0`，先恢复远征 Meta 再恢复活动战斗，并在活动远征中阻止快速读档；旧合约残片和深层材料计数一次性迁移为仓库物品。
- 测试自动发现当前测试文件并显式跳过历史塔防脚本；共 112 项 Node 自动化测试通过。完整说明见 `doc/design/duckov-expedition-phase-one.md`。

### 2026-07-21 · ⚔️ 远征战斗完整调优

- 统一远征移动更新，加入移动射击惩罚、障碍视线与投射物碰撞、远程敌人、冲锋预警和首领范围技，解决双重移动、无限风筝与敌人同质化。
- 重做撤离压力：离圈容错后进度回退、信标争夺暂停、持续追兵和按返程距离增长的威胁；威胁与超限压力同时提高风险和收益。
- 分化三种搜索方式；安全屋改为“消耗补给休整”或“保留补给获取先手”；满包时由玩家手动替换或丢弃，并新增保险格。
- 接入宠物远征经验、等级、羁绊、队伍光环、护卫与濒死救援；补齐玩家生命回复、领地防御与开局属性快照。
- 增加随机地图抖动、迷雾情报保护、一次性世界事件、深层材料、合约残片、撤离红宝石及远征长期里程碑；领地升阶只认成功撤离深度。
- 存档升级至 `1.4.0`，可恢复活动远征的世界、路线、生命、背包、怪物、宠物冷却和撤离进度；页面隐藏时自动保存。
- 新增战斗、远征平衡、宠物成长和世界探索回归，统一测试共 66 项；详见 `doc/design/expedition-combat-audit-fixes.md`。

### 2026-07-14 · 🏆 长期里程碑重做

- 将成就从控制器内的静态奖励列表拆分为独立 `AchievementSystem` 与数据目录，覆盖命运、远征、伙伴、领地四类共 23 项长期里程碑。
- 所有进度按历史最高值保存，并在首次达成时锁存完成时间；消费资源、换下宠物或拆除建筑不会让已达成奖励回退。
- 移除与首局引导和“下一目标”重复的一次性“任务”页，新增分类筛选、可领取优先排序、HUD 数量徽标、实时刷新和全部领取。
- 存档升级到 1.3.0，并自动迁移旧 `ProgressionSystem.claimedAchievementIds`，已领取项目不会重复发奖。
- 增加里程碑完成锁存、批量领取、存档往返、旧领取状态迁移和浏览器交互回归。
- 将成就弹窗重做为“里程碑档案馆”：增加总完成度仪表、分类进度、双色奖章卡、奖励筹码，以及可领取、进行中、已归档三套清晰的视觉状态。

### 2026-07-13 · 🐾 宠物跨场景闭环与进度收敛

- 为 8 只宠物补充探索天赋和基地岗位；天赋只强化匹配搜索方式，并对品质、额外物品、威胁、补给率和伏击率设置统一上限。
- 远征结算会为上阵宠物增加羁绊，成功撤离按深度提供更多成长；重复结算不会重复增加资源或羁绊。
- 基地活动接入宠物岗位，匹配设施时按羁绊提高一次性战备或主动开采奖励；宠物页面会展示天赋与岗位。
- 命运桌和领地 HUD 改用同一个长期目标模型，移除把成长共鸣误写为永久开放条件的旧提示。
- 领地存档升级为版本 3；旧 `unlockedSlots` 不再直接映射最高领地等级，版本 2 的异常高等级会保留为历史资历并按建筑证据保守迁移。
- 修复领地 Canvas 被移除后 `ResizeObserver.observe()` 仍尝试观察空节点的问题，并补充浏览器生命周期回归。
- 羁绊改为只由远征结算提升；基地岗位使用熟悉、默契、挚友三个整数奖励档位，满羁绊按实际增长反馈。
- 下一目标区分可执行、进行中、资源阻塞和可选状态，显示精确资源缺口；零散生产不再抢占核心进度，并提供可点击场景入口。
- 搜索按钮、宠物编队与背包、基地设施面板和远征结算都展示当前实际生效的宠物效果。

### 2026-07-13 · 🏛️ 领地建筑美术升级

- 重制领地完整视觉资源：天空远景、地表道路、分区路标、灯具、施工台、远征门、边界障碍和七类建筑全部使用同一套蓝昏边境风格。
- Canvas 场景不再依赖矩形山脉、纯色地块、代码灯柱、虚线施工框或上一批宫殿式建筑，并保留邻近高亮、等级和生产提示。

### 2026-07-12 · 🏕️ 领地实际基地世界

- 领地由网格管理界面重做为可自由横向移动的 Canvas 基地，支持 A/D、方向键、屏幕按键与近距交互。
- 新增跟随相机、远近景分层、固定功能分区、施工遗迹、远征入口、宠物跟随和场景活动表现。
- 领地成长改为永久 R0–R5 等级；远征深度、成功撤离、建设度和资源共同决定升阶，已开放空间不会回锁。
- 七类建筑改为固定且唯一的基地设施，建筑升级上限随领地等级提升，并保留旧存档的建筑迁移与去重。
- 训练、守护、补给、演练和研究可生成下一局一次性战备；永久建筑加成已接入远征攻击、防御、补给、宠物技能、经验及结算资源。
- 工坊和水晶矿改为受仓储时长限制的主动收取生产，并新增基地世界与生产、活动、升阶的自动测试。

### 2026-07-12 · 🗺️ 宠物远征可操控大地图

- 远征由路线按钮推进升级为可自由行进的大地图，支持 WASD、方向键和屏幕方向键移动。
- 新增跟随相机、世界障碍与碰撞；玩家需要绕开岩石、水域和遗迹，亲自前往事件地点。
- 地图地点支持点击追踪并显示距离；追踪不会触发自动寻路，靠近后需按 `E` 或点击“交互”进入。
- 新增小地图与探索迷雾，显示已揭开区域、玩家位置、已发现地点、追踪目标和入口撤离信标。
- 战斗改为地图内近距自动交战；主角和宠物会自动攻击范围内敌人，玩家仍可移动走位、锁定目标并释放宠物技能。
- 撤离信标固定在地图西侧入口。清理至少 3 个地点后需返回入口启动撤离；离开信标范围时撤离倒计时暂停，重新入圈后继续。
- 远征终端升级为地点追踪和附近交互提示，同时保留搜索选择、补给、背包、威胁与结算信息。
- 补齐入口信标追踪、真实小地图迷雾、弹窗输入隔离，以及新一局宠物位置重置等边界保护。

### 2026-07-11 · ⚔️ 宠物远征·单局 RPG 搜打撤

- 战斗模块重做为单局远征：选择路线、搜索或交战，并在收益与风险之间决定继续深入还是撤离。
- 新增 8 区域远征路线，以及补给站、密封仓库、巡逻区、精英巢穴、安全屋和核心守卫等节点。
- 新增快速搜索、仔细搜刮和宠物侦察三种搜索方式；不同方式拥有不同的收益、威胁和伏击风险。
- 新增本局生命、补给、威胁和 8 格战利品背包。背包满时会自动保留价值更高的物品。
- 主角和上阵宠物会自动战斗；支持点击敌人锁定目标，并手动释放带冷却时间的宠物主动技能。
- 清理 3 个区域后可以主动撤离；启动信标后需要守住倒计时，高威胁会增加撤离压力。
- 撤离成功会带回全部背包和战斗收益；战败或放弃会遗失背包，仅结算部分金币和经验。
- 新增远征终端、路线选择、区域情报、战利品列表、技能栏、撤离提示和自动保存结算反馈。
- 新增独立的远征规则模块与设计说明 `doc/design/extraction-rpg-rework.md`。

## [1.0.1] - 2025-11-29

### 🐛 Bug 修复

#### 严重问题修复
- **修复游戏初始化失败问题**: 修正了 `UISystem` 实例化方式错误，从错误的带参数实例化改为使用单例模式
- **修复 `updateAllAttributeDisplays` 未定义错误**: 该方法在 `player-system.js` 中被调用但未定义，导致加载存档时报错
- **修复存档玩家等级显示问题**: 存档槽位显示的玩家等级始终为 1，现已修正数据结构读取逻辑

#### 中等问题修复
- **修复宠物系统获取玩家等级失败**: `getPlayerLevel()` 方法无法正确获取玩家等级，影响宠物解锁判断
- **修复 `UISystem` 缺少 `update()` 方法**: 游戏核心循环调用了不存在的方法

#### 低等问题修复
- **修复 CSS 类名兼容性问题**: 将中文类名 `养成-bars` 改为英文 `pet-status-bars`

### 🔧 技术改进
- 优化了 `UISystem` 的初始化流程
- 改进了存档数据结构，在顶层添加 `level` 字段便于读取
- 增强了 `getPlayerLevel()` 的容错能力，支持从多个来源获取玩家等级

## [1.0.0] - 2025-11-29

### ✨ 初始版本

#### 核心功能
- 🎮 **游戏核心系统**: Canvas 渲染、游戏循环、场景管理

- 主游戏界面与底部导航栏
- 角色管理界面（角色/技能/宠物/藏品/契约标签）
- 存档管理界面
- 设置菜单

- `F5` - 快速保存
- `F9` - 快速加载
