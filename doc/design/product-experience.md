# 产品体验设计

## 定位

PetPlan 是横屏街机风格的放置成长游戏。屏幕使用统一的像素边框、中央玩法舞台和右侧管理栏：左侧 HUD 用于切换，中央传达当前循环，右侧承载可执行成长项。

## 玩家路径

```text
命运翻转 -> 自动化与训练 -> 战斗收益 -> 领地建设 -> 解锁更高阶段
```

命运场景是默认入口。推荐商店和“下一目标”负责解释当前最有价值的行动，避免玩家在多条成长路径中失去方向。

## 下一阶段：首局与成长方向

首局不引入独立教程页，而是在命运桌上持续显示当前步骤。它是派生状态，不会阻断玩家自由操作：

1. 累计翻转 8 次，理解正面与反面的来源。
2. 将桌面扩充到 2 枚硬币，理解正面成长。
3. 招募第 1 个小助手，理解自动化。
4. 建造第 1 座领地建筑，理解长期目标。

达到前一项后自动显示下一项。目标完成本身不发放新货币，奖励仍由成就页面统一管理，避免新手引导和奖励系统重复结算。

## 成长倾向

游戏根据已投入的升级计算当前领先倾向，不要求玩家在开局做不可逆选择。商店推荐会给领先倾向一个小幅权重，但不会隐藏其他玩法。

| 倾向 | 主要投入 | 体验承诺 |
| --- | --- | --- |
| 先攻 | 手动点击、主角训练 | 更快的主角战斗成长 |
| 协同 | 小助手、结算芯片、助手速度、宠物训练 | 更高的自动结算与宠物成长 |
| 拓域 | 桌面硬币、领地建筑、扩张 | 更快推动循环脉冲与建筑开放 |

倾向只影响推荐排序和可视化，不改写原有成本，也不形成硬锁或重置成本。

## 领地脉冲说明

循环脉冲必须展示其构成：翻转次数、桌面硬币、小助手、主角训练、上阵宠物、建筑和扩张分别贡献多少。领地侧栏仍以“下一建筑”为主，构成信息用于解释进度而不是制造第二个管理面板。

## 奖励语义

成就和任务达到条件后显示“可领取”。玩家主动领取后，奖励才会写入资源系统，并将领取 ID 保存进存档。已领取目标显示“已领取”，不能重复获得。

## 场景与状态

| 场景 | 主要操作 | 进入条件 |
| --- | --- | --- |
| 命运 | 翻转硬币、购买自动化、训练 | 默认入口 |
| 战斗 | 观察自动战斗、升级属性 | 由命运成长推动 |
| 领地 | 建造、升级、收取产出 | 由循环脉冲开放 |
| 宠物 | 编队、查看背包与图鉴 | HUD 模态 |
| 成就 | 查看里程碑与任务 | HUD 模态 |

## 屏幕策略

游戏只支持横屏 16:9 舞台。竖屏显示明确的旋转提示，而不是压缩 HUD、商店与交互目标。桌面和横屏移动端共享同一信息层级，随空间缩放但不改变操作位置。

## 无障碍基线

- 所有 HUD 场景按钮都有可读名称。
- 宠物、成就、玩家、设置模态使用对话框语义。
- 打开模态时焦点进入关闭控件，Tab 键不会离开模态，关闭后返回触发控件。
