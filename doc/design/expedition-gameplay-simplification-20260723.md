# 远征玩法精简改版（2026-07-23）

## 改版目标

本轮不继续增加新系统，而是把单局体验收拢到三个持续发生的决定：

1. 去哪里；
2. 拿多少；
3. 何时撤离。

核心操作目标为：移动、瞄准射击、情境交互、一个队伍技能和暂停退出。

## 本轮范围

- 出发前选择武器，进入远征后锁定本局武器；
- 收拢移动射击惩罚并重新校准三种武器的基础强度；
- 移除玩家等级造成的怪物生命反向膨胀；
- 搜索地点改为一次交互，只保留快速拿取和彻底搜刮；
- 宠物侦察改为搜索方式上的被动天赋；
- 威胁收拢为 0～100 的单一警戒值；
- 满包时自动保留更有价值且未受保护的物品，减少连续弹窗；
- 活动局只提供一个队长宠物主动技能；
- 战斗与撤离阶段主动退出按战败处理，安全阶段才享有主动止损；
- 简化撤离圈规则与双撤离点的定位：离圈只暂停进度，返回后继续，不再回退；
- 行动界面使用全宽战场；整备/结算居中，搜索/营地按情境出现，背包按需打开，不再保留右侧终端。

## 兼容原则

- 旧存档中的武器、搜索、威胁和背包字段仍可读取；
- 旧 `pet` 搜索方式映射到彻底搜刮，不导致活动存档无法恢复；
- 旧超限压力只用于迁移，不再作为新的独立玩法指标累积；
- 旧待选战利品仍允许玩家完成一次手动处理，新掉落采用自动整理。

## 验收目标

- 每个搜索地点只发生一次模式选择和一次结算；
- 活动局不能切换出发武器；
- 同层敌人不会因为玩家升级而额外增加生命；
- 常规掉落不会连续阻断移动和撤离；
- 玩家只需理解一条警戒值；
- 撤离离圈后进度暂停且不回退，增援仍持续抵达；
- 完整 Node 测试和浏览器冒烟测试通过。

## 验证结果

- `npm test`：170 项 Node 自动化测试通过；
- `tests/browser-smoke.html`：31 项浏览器集成检查通过；
- `git diff --check`：通过。
