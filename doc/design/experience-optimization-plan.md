# 核心体验优化方案

> 历史方案：本文记录 2026-07-23 早期的“可折叠右侧终端”设计。远征终端随后被彻底移除，当前实现见 [远征情境交互改版](expedition-context-interaction-20260723.md)。竖屏引导与命运推荐部分仍保留背景参考价值。

## 目标

本方案解决最新版体验复评中确认的四个现存问题，并记录新手引导的下线决定：

1. 远征终端继承旧滚动位置，重新进入或阶段切换后丢失当前任务上下文。
2. 远征进行中 HUD 与终端同时展示过多信息，地图、操作和风险判断互相争夺注意力。
3. 竖屏把 16:9 舞台压缩成不可读区域，却仍表现为可以正常游玩。
4. 命运商店只有两条推荐时，下半部分形成没有信息价值的大面积空区。

优化不改变核心玩法、数值、存档和场景入口，只重排信息层级、状态反馈与响应式行为。

## 体验原则

- **当前任务始终可见**：进入场景或阶段变化后，玩家无需滚动寻找下一步。
- **按阶段揭示信息**：整备、探索、搜索、战斗、撤离和结算使用不同信息密度。
- **不以缩小字号换取适配**：空间不足时折叠次要信息，而不是把完整桌面界面压小。
- **视觉行为一致**：视觉上遮罩全屏的界面必须提供明确的焦点、关闭和恢复路径。
- **空白承担解释价值**：推荐列表不足以填满侧栏时，用下一里程碑和推荐理由补足决策信息。

## 1. 远征终端：阶段化上下文坞

### 1.1 信息分层

远征终端分为三层：

| 层级 | 内容 | 滚动行为 |
| --- | --- | --- |
| 固定上下文头 | 当前阶段、区域名称、关键风险、主要动作 | 始终可见，不参与滚动 |
| 当前阶段区 | 路线、搜索选择、撤离选择或结算 | 阶段变化时自动定位到顶部 |
| 次要管理区 | 配装、仓库、委托、详细风险、背包明细 | 默认折叠或位于当前阶段区之后 |

### 1.2 各阶段默认内容

| 阶段 | 默认展开 | 默认收起 |
| --- | --- | --- |
| `briefing` | 配装、委托、风险预估、开始远征 | 背包历史 |
| `route` | 当前目标、路线选择、背包摘要 | 完整配装、委托、详细风险 |
| `search` | 搜索方式、搜索进度、背包摘要 | 配装、委托 |
| `camp` | 休整选择、补给状态 | 配装、委托 |
| `combat` | 当前威胁、敌人、补给、放弃 | 路线、配装、委托 |
| `extraction-ready` | 两种撤离方案及成本 | 配装、委托 |
| `extracting` | 守点进度、圈内状态、敌人压力 | 路线、配装、委托 |
| `extracted` / `defeat` | 结算与返回整备 | 其余全部 |

### 1.3 滚动规则

- 首次进入远征、从其他场景返回远征时，将终端定位到当前阶段区顶部。
- 阶段发生变化时定位到新阶段区顶部，并通过礼貌级实时区域宣布阶段名称。
- 同一阶段内因数值刷新、移动或战斗帧更新，不改变玩家手动滚动位置。
- 当前阶段摘要使用 `position: sticky`，即使玩家查看次要信息也不会丢失主要动作。
- 自动滚动只在“进入场景”或“阶段变化”两个离散事件发生，避免每帧抢夺控制权。

## 2. 远征进行中 HUD：三层战斗信息

### 2.1 常驻层

地图上只常驻会立即改变操作决策的信息：

- 生命与宠物护卫合并为生存组。
- 弹药与换弹状态合并为武器组。
- 当前目标、距离和交互状态合并为目标组。
- 威胁、背包和探索深度合并为风险组。
- 敌人数只在战斗或撤离守点阶段出现。

金币、红宝石、水晶、战力、成功撤离次数、本局击杀和携带价值保留在终端，不再与战斗地图争夺首屏空间。

### 2.2 情境操作层

- 探索时：方向键、交互键、当前追踪目标。
- 战斗时：方向键、射击、换弹、宠物技能；交互键隐藏。
- 搜索时：搜索进度和取消；射击键仅在伏击发生后出现。
- 撤离时：守点进度、圈内/离圈状态、射击和技能。

### 2.3 终端管理层

完整路线、风险结算、配装、仓库、委托和背包明细只在右侧终端中出现。窄横屏下，进行中终端默认缩为“任务抽屉”，点击后覆盖地图右侧；整备与结算阶段保持展开。

## 3. 竖屏：软旋转引导

PetPlan 当前正式支持横屏 16:9。竖屏不继续压缩完整舞台，也不建设第二套高成本战斗布局，而是显示可访问的软旋转引导：

- 标题：“横屏展开完整远征”。
- 说明：“旋转设备后会保留当前进度与操作位置。”
- 展示三个横屏收益：地图更清楚、操作区完整、终端可读。
- 提供“我已旋转”按钮，用于重新检测方向并把焦点交回游戏。
- 屏幕方向变化后自动隐藏引导，不刷新页面、不重置远征。
- 引导使用 `role="dialog"`、`aria-modal="true"`，打开时聚焦主按钮；Escape 不关闭，因为竖屏状态下继续游戏不可用。
- 宽度不超过 540px 的竖屏设备显示引导；桌面窄高窗口同样适用。

后续只有当竖屏用户占比和远征完成率证明值得投入时，才进入“地图全宽 + 底部抽屉”的原生竖屏二期。

## 4. 新手引导：当前版本下线

当前版本移除 HUD“指引”入口、教练浮层、高亮焦点环和运行时控制器。底层成长进度、存档兼容字段与“下一目标”继续保留，避免影响旧存档、成长推荐和长期目标；后续如重新设计引导，应作为独立方案重新评估，而不是恢复旧浮层。

## 5. 命运推荐栏：下一里程碑卡

推荐筛选只显示两项时，在列表尾部加入非购买型“下一里程碑”卡：

- 当前目标标题与精确缺口。
- 主推荐的原因。
- 备选成长方向。
- 若目标位于远征或领地，提供场景跳转按钮。

该卡不参与商店排序、不消耗资源、不复制成就奖励，只解释“为什么现在推荐这两项”。其他筛选拥有三项或以上商品时不显示，避免挤压购买列表。

## 6. 响应式规格

| 视口 | 远征终端 | 战斗 HUD | 命运推荐 |
| --- | --- | --- | --- |
| ≥ 1100px 横屏 | 固定右侧上下文坞 | 完整常驻层 | 两卡 + 里程碑卡 |
| 721～1099px 横屏 | 较窄上下文坞 | 次要文本缩略 | 紧凑里程碑卡 |
| ≤ 720px 或高度 ≤ 420px 横屏 | 进行中任务抽屉；整备展开 | 44px 以上触控目标 | 隐藏里程碑长说明 |
| 竖屏 | 显示软旋转引导 | 不渲染为可操作舞台 | 不单独展示 |

## 7. 验收标准

### 远征

- 从命运或领地返回远征时，终端当前阶段标题和主要动作在首屏可见。
- 同一阶段内手动滚动后，常规状态刷新不会改变 `scrollTop`。
- 阶段变化后终端只自动定位一次。
- 战斗阶段地图上不再同时展示局外货币、战力和六项完整统计。
- 844×390 下移动、射击、换弹与导航触控目标不小于 44×44px。

### 竖屏

- 390×844 下不出现被压缩成 82px 宽的地图区域。
- 旋转引导文案和主按钮无需缩放即可阅读与操作。
- 切换回横屏后当前场景和远征状态保持不变。

### 新手引导下线

- HUD 不再显示“指引”入口。
- 页面不再创建教练浮层、高亮焦点环或新手引导控制器。
- 旧存档中的成长进度字段仍可读取，不影响“下一目标”和推荐排序。

### 命运推荐

- 推荐筛选为两张商品卡时出现下一里程碑卡。
- 切换到命运、自动或成长筛选时里程碑卡隐藏。
- 卡片内容随统一下一目标和推荐结果刷新。

## 8. 验证指标

- 返回远征后 5 秒内产生有效移动或阶段操作的比例。
- 首次远征中打开终端完整配装/委托区的次数与阶段分布。
- 首次完成一个区域、解锁撤离、成功撤离的转化率。
- 竖屏进入后 30 秒内转为横屏的比例。
- 推荐卡点击率与下一里程碑场景跳转率。
