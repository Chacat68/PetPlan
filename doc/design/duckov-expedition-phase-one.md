# 宠物远征参考搜打撤玩法：第一阶段修改说明

> 历史阶段记录：当前玩法规则以 [远征玩法精简改版（2026-07-23）](./expedition-gameplay-simplification-20260723.md) 为准。

> 完成日期：2026-07-21
> 存档版本：`1.5.0`
> 参考方向：借鉴《逃离鸭科夫》式“搜索—交战—撤离—整备”节奏，不复制其地图、美术、角色或具体数值。

## 1. 本阶段目标

本轮把原有“大地图自动交战远征”推进为可操作的轻量搜打撤：

1. 玩家亲自瞄准、射击、切枪和换弹，命中由固定弹道与走位决定。
2. 搜索改为地图实体容器与计时读条，风险发生在搜索过程中，而不是点击后瞬间开奖。
3. 撤离不再只有单一路径，入口撤离与深区应急撤离形成返程距离和守点压力取舍。
4. 战利品不再自动折现，成功带回后进入仓库，用于配装、合约或出售。
5. 建立“整备—出发—带回/遗失—再次整备”的局内外闭环。

## 2. 当前核心循环

```text
仓库配装与接合约
  → 进入大地图
  → 追踪并探索 POI
  → 搜容器 / 遭遇战 / 安全屋
  → 背包取舍与保险
  → 入口撤离或深区应急撤离
  → 战斗收益结算、物品入库、合约推进
  → 重新配装
```

远征失败时，背包中只有玩家显式放入保险格的物品可回收；系统不再自动保护最高价值物品。非永久护甲和宠物链接器也会承担失败遗失风险。

## 3. 手动枪械战斗

### 3.1 操作

| 操作 | 桌面 | 触屏 |
| --- | --- | --- |
| 移动 | `WASD` / 方向键 | 屏幕方向键 |
| 瞄准 | 鼠标移动 | Canvas 拖动 |
| 射击 | 按住 Canvas 左键 | 按住开火键 |
| 切枪 | `1` / `2` / `3` | 武器模式按钮 |
| 换弹 | `R` / 换弹按钮 | 换弹按钮 |
| 交互 | `E` | 交互按钮 |

Canvas 显示短瞄准线与准星。玩家弹丸沿开火时的固定方向飞行，不会追踪敌人，允许打空；敌方远程攻击也改为可见弹丸，可以侧移躲避或利用障碍阻挡。

### 3.2 武器定位

| 武器 | 定位 | 主要代价 |
| --- | --- | --- |
| 巡林步枪 | 中距离稳定持续输出 | 单发爆发较低 |
| 拓荒霰弹枪 | 近距离多弹丸爆发 | 射程短、移动散布大 |
| 哨卫精确枪 | 远距离高单发伤害 | 弹匣小、射速和换弹慢 |

三种武器各自保存弹匣、备弹和换弹进度。永久攻击属性只经过对数软缩放参与枪械伤害，避免旧高等级存档跳过交火过程。移动射击会扩大散布、降低伤害并拖慢射击节奏。

## 4. 计时搜索与实体容器

- 普通搜索地点生成 2 个容器，密封仓库生成 3 个。
- 玩家必须走到容器附近才能开始搜索。
- 快速搜索、仔细搜刮和宠物侦察继续保持不同耗时、收益、补给成本与伏击概率。
- 读条完成前不会提前生成物品、扣补给、增加威胁或判定伏击。
- 移动、受伤或主动取消会中断搜索并释放容器，不产生任何结算。
- 容器状态、当前读条与进度均可保存恢复，全部容器完成后地点才算清理。
- 侦察站的视野强化会真实扩大一次地图揭示范围。

## 5. 双撤离点

| 撤离点 | 解锁 | 成本 | 特征 |
| --- | --- | --- | --- |
| 入口撤离信标 | 深度至少 3 | 无 | 需要返程，守点压力较低 |
| 深区应急撤离点 | 深度至少 5 | 1 份补给 | 少走返程，但守点更久、敌人和增援更多 |

两处撤离点分别保存锁定、解锁和激活状态。补给事件、补给使用、休整、搜索和读档后都会重新同步可用性，避免规则层已解锁但地图仍不可交互。

撤离倒计时只在玩家位于信标圈内且信标未被敌人争夺时推进；离圈超过容错会回退，撤离期间持续出现增援。

## 6. 仓库、配装与合约

### 6.1 仓库与配装

- 仓库默认 40 格，溢出战利品进入待领取箱，不会被吞掉。
- 所有仓库物品均可在滚动列表中操作，不再只显示前几格。
- 永久制式训练武器组不会遗失，可选择本局默认枪械模式。
- 完整机械芯可装备到护甲槽，增加本局防御。
- 秘境水晶和星辉核心可装备到宠物链接器槽，增加本局宠物护盾。
- 精制宠粮可放入四个消耗品槽，出发时消耗一份并转化为额外补给。
- 补给栏满时必须先卸下一件物品，不会静默替换第一格。
- 远征开始后锁定仓库、配装和合约写操作，避免局中转移风险物品。

### 6.2 合约

委托板提供五类目标：带回物资、调查地点、击杀精英、指定宠物撤离和高威胁撤离。最多同时接受 1 条主线和 2 条支线；已满类别在界面直接显示“槽位已满”。

合约进度由实际远征结算推进。交付物资会从仓库扣除，奖励进入资源系统或待领取箱。结算使用持久化 raid ID，跨重启也不会把新一局误判为重复结算。

## 7. 经济与结算边界

| 结果 | 战斗现金收益 | 背包物品 | 非永久配装 |
| --- | --- | --- | --- |
| 成功撤离 | 全额 | 全部入仓 | 保留 |
| 战败 | 金币 30%、经验 40%、水晶 0 | 仅保险格入仓 | 未投保则遗失 |
| 主动放弃 | 金币 12%、经验 20%、水晶 0 | 仅保险格入仓 | 未投保则遗失 |

背包物品的金币、水晶和经验字段只用于价值预览与出售参考，不会在入仓时再次直接结算。玩家可选择后续出售、装备或交付合约，从而消除“物品入库一次、货币又自动发一次”的双重收益。

## 8. 存档与迁移

- `SaveSystem` 升级至 `1.5.0`，新增 `expeditionMeta` 数据域。
- 加载顺序先恢复 Meta，再恢复活动战斗，保证配装快照和 raid ID 一致。
- 保存活动局的枪械、弹药、换弹、准星、固定弹丸、容器读条、双撤离状态和配装效果。
- 远征进行中禁止快速读档，避免通过旧存档复制战利品。
- 旧 `contractFragments` 和 `deepMaterials` 计数会迁移为真实仓库物品，迁移只执行一次。
- 仓库领取即使只成功转入一部分，也会立即标记变更并保存。

## 9. 主要文件

- `js/modules/combat-system.js`：手动枪械、敌方弹丸、配装效果、搜索/撤离接线与局内存档。
- `js/modules/expedition-run-system.js`：计时搜索、掉落、背包、保险、双撤离与结算规则。
- `js/modules/expedition-world-system.js`：实体容器、世界事件、双撤离点和地图状态。
- `js/modules/expedition-meta-system.js`：仓库、待领取箱、配装、合约、迁移和幂等账本。
- `js/controllers/battle-scene-controller.js`：鼠标/触屏战斗操作、仓库配装、合约和状态展示。
- `js/modules/save-system.js`、`js/main.js`：`1.5.0` 保存、加载顺序与系统组装。

## 10. 验证结果

执行：

```powershell
npm test
```

当前 112 项 Node 自动化测试全部通过，覆盖手动枪械、固定弹道、敌方弹丸、计时搜索、容器存档、双撤离、失败保险、仓库溢出、配装效果、带配装活动局恢复、合约推进、结算幂等、Meta 迁移和快速读档保护。旧塔防脚本作为历史诊断页显式跳过，不参与当前玩法验收。

建议浏览器手工复核三项观感：准星与移动散布是否清晰、触屏拖动瞄准是否顺手、40 格仓库滚动与按钮密度是否合适。这些是交互体验校准，不影响当前规则闭环。

## 11. 下一阶段建议

下一阶段优先依据实玩数据处理，而不是继续盲目堆系统：

1. 记录搜索取消率、武器使用率、弹药耗尽率、两种撤离选择率和配装损失率。
2. 为敌人增加听觉/视野警戒、脱战与巡逻状态，让“避战”成为可读策略。
3. 增加武器配件、弹药类型和更明确的装备品质，但保持每件物品有清楚用途。
4. 让地图情报、钥匙和封锁区形成跨局准备，而不是只提高数值。
5. 根据失败原因区分教学提示，帮助玩家理解是弹药、路线、威胁还是撤离时机导致损失。
