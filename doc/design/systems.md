# 游戏系统设计

## 命运系统

命运桌是起始与自动化成长系统。玩家翻转桌面硬币获得正面或反面资源；商店会按当前进度给出推荐。

- 正面、反面：命运商店的主要成本。
- 桌面硬币：增加可点击目标与命运循环容量。
- 小助手和结算芯片：提高自动结算频率与收益。
- 主角训练、宠物训练：把命运资源转成战斗属性。

自动结算通过 `FateCoinSystem` 触发 UI 回调，保持资源计算与桌面动画分离。

命运桌还派生两类体验状态：首局四步目标和当前成长倾向。目标按翻转、第二枚硬币、首个助手、首座建筑依次显示；倾向由已投入的先攻、协同、拓域相关升级计算，仅轻度影响推荐排序。

## 战斗系统

战斗使用 Canvas 持续渲染主角、宠物、怪物与投射物。战斗只在战斗场景运行，离开时暂停。

- 主角属性：攻击、生命、恢复、暴击、攻击速度、多重射击。
- 宠物：最多 3 只上阵，提供协战和属性加成。
- 奖励：金币、红宝石、水晶，用于宠物解锁与领地建设。

## 宠物与成就

宠物页面支持编队、背包和图鉴；成就页面展示长期里程碑与任务。两者均为主场景上的模态管理页，不会切断当前进度。

成就与任务奖励需要玩家点击领取。领取后，奖励会进入资源系统，并由 `ProgressionSystem` 记录已领取 ID；相同目标不会重复发放。

## 领地系统

领地把命运、战斗和宠物进度汇总成长期目标。

- 初始只有 1 个开放地块，最多 12 个。
- 地块与建筑开放由命运循环脉冲及跨系统进度推进，而不是单纯玩家等级。
- 主基地、训练场、神庙、兵营、工坊、水晶矿、图书馆依次成为建筑目标。
- 工坊和水晶矿提供按节拍收取的资源产出。

领地只由主游戏中的 `TerritorySystem` 管理。旧 `territory.html` 链接会重定向至 `index.html?scene=territory`。

脉冲数值由翻转、桌面硬币、小助手、主角训练、宠物、建筑和扩张的固定权重相加。`TerritorySystem` 会同时返回总脉冲和逐项构成，领地 UI 显示这些贡献以说明下一建筑的解锁原因。

## 资源与存档

资源系统管理金币、红宝石和水晶。`SaveSystem` 将玩家、资源、战斗、宠物、领地和命运状态写入 LocalStorage 的存档槽位；槽位 1 每 30 秒自动保存。

## 体验约束

- 视觉与交互以横屏 16:9 舞台为基准。
- 竖屏设备显示旋转提示，不缩放为不可操作的横屏画面。
- 模态打开后焦点停留在模态内，关闭后回到原触发控件。
