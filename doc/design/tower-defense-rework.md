# 宠物防线：竖版塔防改造说明

> **历史版本说明（2026-07-11）**
>
> 本文记录的“宠物防线”塔防玩法已被“宠物远征”单局 RPG 搜打撤模式替代，不再代表当前战斗规则。现行规则与界面分别见 [远征玩法精简改版](expedition-gameplay-simplification-20260723.md) 和 [远征情境交互改版](expedition-context-interaction-20260723.md)；初版搜打撤说明也已转为历史记录。本文保留用于追溯项目演进与旧存档字段来源。

## 1. 改造结论

原有“主角与宠物横向追击、怪物持续随机刷新”的战斗，已改造成可完整游玩的“宠物防线”塔防模式。

这里的“竖版”指横屏游戏舞台内的纵向战场：敌人从画面顶部出现，沿 S 形路线向底部基地推进。项目原有的横屏方向锁、命运桌、宠物编队、领地和存档入口均保留。

本次交付包含：

- 10 波固定流程与第 10 波 Boss。
- 纵向标准化路径与漏怪扣基地生命。
- 主角固定为底部英雄塔。
- 最多 3 只上阵宠物自动部署为宠物塔。
- 6 个固定塔位，波次间可点击换位。
- 宠物塔锁敌、投射物、范围伤害、减速、连锁、眩晕、治疗和斩杀效果。
- 局内战斗能量、宠物塔升级和基地修复。
- 胜利、失败、一次性奖励结算及玩家经验升级。
- 最佳波次、胜利次数和失败次数存档。
- 新的防线指挥 UI、自动化测试与浏览器验收。

## 2. 一局战斗流程

1. 进入“战斗”，系统读取当前主角属性、领地加成和上阵宠物，建立本局基地与宠物塔。
2. 点击“开始第一波”。敌人按波次配置从顶部进入，沿路径向基地推进。
3. 主角和宠物塔自动攻击最接近基地的敌人。
4. 击杀敌人获得本局能量，并累计待结算金币、水晶和经验。
5. 一波全部生成且场上敌人清空后进入波次间歇。
6. 间歇期可以升级宠物塔、修复基地，或先点击一个宠物塔再点击空塔位来换位。
7. 守住第 10 波 Boss 后胜利；基地生命降到 0 时失败。
8. 结算完成后点击“重新挑战”开始新的一局。

离开战斗场景只会暂停当前战斗，不会清空波次；返回战斗后继续运行。

## 3. 核心规则与数值

### 3.1 波次

- 总波数：10。
- 第 1～9 波普通敌人数：`5 + 波次 × 2`。
- 第 5 波会出现精英怪。
- 第 10 波：12 只混合敌人，最后生成 Boss 龙。
- 随波次逐步加入蝙蝠、骷髅和恶魔，并提高生命、攻击和少量移动速度。

敌人职责：

| 敌人 | 定位 |
| --- | --- |
| 史莱姆 | 基础小怪 |
| 哥布林 | 均衡小怪 |
| 疾风蝙蝠 | 快速怪 |
| 重甲骷髅 | 高生命怪 |
| 深渊恶魔 | 后期重型怪 |
| 防线终结者 | 第 10 波 Boss |

### 3.2 基地

基地拥有独立的局内生命，不会直接改写玩家永久生命：

```text
基地最大生命 = 180 + 玩家最大生命 × 1.4 +（玩家防御 + 领地防御）× 8
最低基地生命 = 250
```

- 漏怪根据怪物攻击力扣除基地生命。
- 玩家防御和领地防御会降低漏怪伤害。
- 玩家生命恢复属性每秒恢复等量基地生命。
- 修复基地消耗 35 点战斗能量，恢复基地最大生命的 22%。

### 3.3 战斗能量

- 开局能量：100。
- 普通击杀：7。
- 精英击杀：15。
- Boss 击杀：35。
- 每波完成奖励：`28 + 当前波次 × 2`。
- 战斗能量仅在本局内使用，重整防线后重置。

宠物塔最高 5 级，升级费用依次为 40、75、110、145 点能量。每级提高伤害，并小幅提高攻击频率。

### 3.4 结算

- 胜利：发放本局累计金币、水晶和经验的 100%。
- 失败：发放金币、经验的 40%，水晶的 25%。
- 领地图书馆的经验百分比加成会作用于最终经验。
- `rewardGranted` 和本局 `settled` 标记保证单只怪物与整局奖励都只结算一次。
- 玩家经验达到升级需求后自动升级；每级同时获得 2 攻击和 5 最大生命。

## 4. 宠物塔职业

宠物仍使用原有拥有、等级和上阵数据，不增加存档迁移成本。

| 宠物类型 | 塔防定位 | 主要效果 |
| --- | --- | --- |
| 火焰犬 | 烈焰炮台 | 范围爆炸 |
| 冰霜猫 | 寒霜控制 | 伤害并减速 |
| 雷电鸟 | 连锁闪电 | 命中后跳跃多个目标 |
| 大地熊 | 大地震击 | 高伤害并短暂眩晕 |
| 风暴龙 | 风暴穿透 | 远射程、快速攻击和连锁 |
| 光明独角兽 | 圣光支援 | 攻击时治疗基地 |
| 暗影狼 | 暗影收割 | 对低生命敌人造成斩杀增伤 |
| 凤凰 | 涅槃火雨 | 大范围高伤害爆炸 |

当前最多上阵 3 只宠物，因此 6 个塔位用于阵型调整，并为后续扩编预留空间。新增或更换上阵宠物后，战斗系统会按宠物实例 ID 同步塔位，不改写宠物实例数据。

## 5. 操作说明

- “开始第一波 / 开始第 N 波”：启动下一波。
- 点击有宠物的圆形塔位：选择该宠物塔并显示属性与攻击范围。
- 波次间先点击宠物塔，再点击空塔位：移动宠物塔。
- “升级宠物塔”：消耗能量提升当前选中塔。
- “修复基地”：消耗 35 能量恢复基地生命。
- “重整防线 / 重新挑战”：放弃当前局内状态并建立新的一局。

主角永久升级仍位于玩家信息弹窗；战斗侧栏只处理局内塔防操作。

## 6. 代码结构

### 新增

- `js/modules/wave-system.js`
  - 生成 10 波敌人计划。
  - 控制刷怪间隔。
  - 管理 `ready / spawning / combat / intermission / victory` 状态。
- `tests/tower-defense-smoke.mjs`
  - 覆盖波次、路径、漏怪、塔升级、奖励幂等和中心碰撞。
- `doc/design/tower-defense-rework.md`
  - 本改造说明。

### 主要修改

- `js/modules/combat-system.js`
  - 改为塔防总协调器，管理路径、基地、宠物塔、投射物、效果和结算。
- `js/modules/player-system.js`
  - 主角固定到基地；新增统一经验入口 `addExperience()`。
- `js/modules/pet-system.js`
  - 塔防模式关闭旧巡航/冲刺 AI，改为按塔位渲染宠物攻击动画。
- `js/modules/targeting-system.js`
  - 新增 `path-progress` 锁敌策略，优先攻击最接近基地的目标。
- `js/modules/game-core.js`
  - 使用塔防背景，限制单帧最大时间，并在系统接入时同步地图尺寸。
- `js/main.js`
  - 接入指挥按钮、Canvas 塔位点击、HUD 同步、暂停恢复和结算保存。
- `index.html`、`css/style.css`
  - 将旧战斗升级栏替换为防线指挥面板。

## 7. 存档兼容

- 不保存当前怪物、投射物、波次、基地生命或局内能量；加载存档后会建立一局新的防线。
- 保存 `bestWave`、`victories` 和 `defeats` 三项长期战斗记录。
- 旧存档中的空战斗对象或旧战斗字段可以安全加载。
- 宠物仍只保存原有宠物实例和上阵实例 ID。
- 战斗结算后会主动保存槽位 1，原有 30 秒自动保存仍保留。

## 9. 后续扩展建议

本次版本已形成可玩的塔防闭环，后续适合按以下顺序继续：

1. 增加战前宠物卡拖放部署，而不只使用自动部署与点击换位。
2. 将命运硬币做成每 2～3 波一次的局内祝福三选一。
3. 增加飞行怪、护盾怪、治疗怪和分裂怪。
4. 为精英与 Boss 增加独立技能预警和阶段变化。
5. 增加关卡地图配置，使路线、塔位、波次与奖励可以数据化扩展。
6. 增加速度切换、暂停按钮、战斗统计和首次玩法引导。
