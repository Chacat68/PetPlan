# 远征情境交互改版（2026-07-23）

## 改版目标

本轮只重做交互层，不修改远征掉落、敌人、搜索、警戒和撤离数值。目标是移除长期占据右侧空间的“远征战术终端”，让玩家在行动中始终先看地图和角色，只在真正需要做选择时看到对应操作。

交互被收拢为三种状态：

1. 出发前与结算后：居中面板；
2. 探索与战斗中：全宽战场；
3. 搜索或营地决策：底部情境操作条。

## 阶段交互

| 阶段 | 主要界面 | 玩家操作 |
| --- | --- | --- |
| `briefing` | 居中整备卡 | 选择武器；按需展开配装、仓库与委托；开始远征 |
| `route` | 全宽地图、顶部状态与目标条 | 移动；`M` 或点击目标条轮换追踪；靠近后 `E` 交互 |
| `search` 待选择 | 底部情境条 | 快速拿取或彻底搜刮 |
| `search` 进行中 | 原位搜索进度 | 等待完成、移动/受伤中断或主动取消 |
| `camp` | 底部情境条 | 安全休整或直接离开 |
| `combat` | 战斗 HUD | 移动、射击、`Q` 队伍技、`R` 补给 |
| `extraction-ready` | 全宽地图与目标条 | 追踪撤离点，靠近后 `E` 启动 |
| `extracting` | 战斗 HUD 与 Canvas 撤离进度 | 守在圈内、射击与补给 |
| `extracted` / `defeat` | 居中结算卡 | 查看摘要并返回整备 |

## 按需界面

### 目标条

原路线列表被顶部目标条替代。点击目标条或按 `M`，会在当前路线、已经发现的一次性事件和已经解锁的撤离点之间循环追踪。追踪只改变地图导航，不会自动寻路或直接触发地点。

### 远征背包

顶部“背包”状态本身就是入口。在 `route`、`camp` 和 `extraction-ready` 等安全阶段，点击它或按 `B` 打开轻量抽屉；战斗、搜索和撤离守点期间自动关闭且不能整理安全袋。

常规满包继续自动保留更高价值物品。为兼容旧活动存档，如果仍存在 `pendingLootChoice`，背包抽屉会强制居中并要求先完成取舍，避免旧局卡死。

### 补给与退出

补给不再占用决策面板，条件允许时通过 `R` 或画面内补给按钮使用。退出只保留现有 `Esc` 确认：安全阶段按主动止损结算，战斗与撤离守点中按战败结算，不再提供第二个“放弃本局”按钮。

## 已删除的交互

- 固定右侧终端、终端标题、在线状态和滚动区；
- `Tab / M` 展开或收起终端；
- 常驻路线卡列表和重复的撤离按钮；
- 终端内“放弃本局”二次确认；
- 局内长期展示的配装、仓库、委托、风险和背包明细。

`Tab` 恢复浏览器默认焦点导航；`M` 只负责轮换地图目标。

## 实现边界

- `index.html`：将右侧 `aside` 改为不参与布局的 `battle-flow-layer`，并提供整备卡、情境条和背包抽屉三个稳定节点。
- `css/style.css`：战斗板固定占满场景；交互层使用绝对定位和 `pointer-events` 隔离，隐藏时不拦截 Canvas。
- `battle-scene-controller.js`：移除终端展开状态与 Tab 拦截，新增目标候选轮换、背包安全阶段校验、旧存档强制取舍和阶段情境同步。
- `main.js`：交互层按普通覆盖层显示，不再作为右侧 Flex 栏。

本轮保留原有玩法系统 API，因此搜索、营地、撤离、仓库、合约、保险和结算规则没有复制到表现层。

## 2026-07-24 战斗 HUD 结构整理

- 将品牌/阶段、当前目标和六项局内状态合并为统一的顶部指挥栏，避免同类信息分散在画布四周。
- 将交互、队伍技能、补给和射击收拢为右下角操作区，并在情境决策条出现时自动隐藏，避免重复操作入口互相遮挡。
- 武器与弹药留在左下信息区，事件反馈紧邻画布底边；小尺寸横屏会进一步隐藏次要说明，只保留当前动作。
- 删除左下角屏幕方向键及其指针状态，远征移动统一使用 WASD 或方向键；领地原有屏幕左右键不受影响。
- 保持画布、背包抽屉和所有既有按钮 ID 稳定，不改变搜索、战斗、撤离与存档系统接口。

## 验证结果

- `npm test`：170 项 Node 自动化测试通过；
- `tests/browser-smoke.html`：31 项浏览器集成检查通过；
- 浏览器实测：战斗板占满场景、交互层不参与左右分栏、搜索情境条与背包抽屉可交互、页面无 warning/error；
- `Esc` 取消/确认、活动局保存恢复和旧待选物品兼容链路继续保留。
