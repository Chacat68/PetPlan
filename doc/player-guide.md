# PetPlan 玩家指南

## 开始游戏

PetPlan 使用横屏舞台。请通过本地服务器打开，并在移动设备上旋转为横屏。

默认进入命运场景：中央是桌面硬币，右侧是按当前状态整理的商店，左侧 HUD 用于切换战斗、领地、宠物和成就。

## 命运桌

点击桌面硬币会结算正面或反面。两类资源用于购买不同成长项：

- **购买硬币**：增加桌面可点击目标。
- **结算芯片**：提高自动结算收益。
- **小助手**：自动前往硬币并结算。
- **银色点击**：提高人工点击收益。
- **金色引擎**：缩短自动结算间隔。
- **主角训练 / 宠物训练**：提升后续战斗表现。

优先查看右侧“推荐”和中央“下一目标”。推荐会随资源、自动化与领地进度变化。

首次进入时，命运桌还会依次提示翻转、扩桌、招募助手和建造第一座建筑。它只说明下一步，不会限制你购买其他项目。

已投入的升级会形成“先攻”“协同”或“拓域”倾向。倾向会帮助推荐理解你的打法，但随时可以改投其他方向。

## 战斗

战斗场景会持续运行主角、上阵宠物、怪物和投射物。右侧升级栏可消耗金币提升攻击、生命、恢复、暴击、攻击速度和多重射击。

战斗奖励为金币、红宝石和水晶：

- 金币用于属性升级、宠物和建筑。
- 红宝石用于稀有宠物。
- 水晶用于领地建筑与扩张。

## 宠物

HUD 的“宠物”按钮打开管理页面。

- **编队**：最多上阵 3 只宠物。
- **背包**：查看并上阵已解锁宠物。
- **图鉴**：查看解锁要求与资源成本。

宠物训练由命运商店触发；上阵状态会影响战斗和领地的进度条件。

## 领地

HUD 的“领地”按钮打开主游戏内的领地场景。旧书签或链接访问 `territory.html` 时也会进入这里。

- 开局只有 1 个开放地块，最多可解锁 12 个。
- 地块和建筑受命运循环脉冲与跨系统成长推进。
- 建造或升级建筑会提供属性加成、金币产出、水晶产出或经验加成。
- 工坊与水晶矿的产出可在领地中收取。

领地侧栏会显示当前循环阶段、已开放建筑、下一个建筑和资源库存。

“脉冲构成”会列出翻转、硬币、助手、训练、宠物、建筑和扩张带来的贡献，可用来判断如何最快达成下一个建筑目标。

## 成就

HUD 的“成就”按钮打开里程碑和任务页面。达到目标后会显示“可领取”；点击领取后，资源立即进入库存，状态变为“已领取”。

## 存档与键盘

| 输入 | 行为 |
| --- | --- |
| F5 | 保存到槽位 1 |
| F9 | 加载槽位 1 |
| Escape | 关闭当前弹窗 |
| Tab | 在打开的弹窗内移动焦点 |

存档保存在浏览器 LocalStorage。清除站点数据会清空进度；请在设置页面定期导出备份。

## 常见问题

**游戏无法启动**：不要直接双击 HTML。运行 `python3 -m http.server 4174 --directory .` 后访问 `http://localhost:4174`。

**竖屏显示提示**：游戏目前只支持横屏。旋转设备后会自动进入游戏。

**领地资源不一致**：所有领地进度都以主游戏内的领地场景为准。旧 `territory.html` 只负责跳转。
